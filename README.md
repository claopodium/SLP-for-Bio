
## 数学建模与基本假设
蛋白质序列 $x =[x_1,x_2,..., x_l]$ 的各个位点独立作用于表型$y$，定义$z$为

$$z=\theta_0+[\theta_1,\theta_2,...,\theta_l] [x_1,x_2,...,x_l]^\mathrm{T}$$

$$\theta_i= [\theta_{i1},\theta_{i2},...,\theta_{ic}]$$

$$x_i=[x_{i1},\ x_{i2},...,\ x_{ic}]^\mathrm{T}$$

其中 $c$ 为序列 $x_i$ 的某位点所有可能事件的数量（如蛋白质有20种氨基酸， $c$ 为 $20$ ）， $l$ 为序列长度。  $x_i$为one-hot向量，形如 $\left[0,\ 0,\ \ldots,1\ ,\ \ldots,\ 0\right]^T$ ，因此权重 $\theta_{ij}$ 直接反映了 $i$ 位置第 $j$ 种事件对表型 $y$ 的影响。最后再加Sigmoid函数，预测值

$$\hat{y}=\frac{e^z}{e^z+1}$$

二分类任务（表型只有界限分明的2种）中，损失函数为交叉熵损失，即
$$\mathcal{L}(\theta)=\sum_{n} y_i\log{\hat{y}_i}+(1-y_i)\ log (1-\hat{y}_i)$$

回归任务则主要基于极大似然估计，损失函数为MSE
$$\mathcal{L}(\theta)=\sum_n (y - \hat{y})^2$$

## 全变分正则化项
向损失函数中引入全变分正则化项 $L_{tv}$ 。生物学解释为，由于蛋白质的相邻残基往往属于同一结构域，故二者常具有连续性，其重要性（参数）相比于更远的残基之间倾向于差别不太大。TV正则化定义为

$$L_{tv}=\lambda\sum_{n}\sum_{i=1}^{l-1}{|\sum_{j=1}^{c}{{(\theta}_{i+1,j}-\theta_{i,j})|}}$$

## 数据格式和使用方法

### 二分类任务

假设目前有A、B两种表型的序列数据，需要准备三份.fasta文件，分别为野生型的序列，所有正例（表型A）的序列放在一个.fasta文件内，所有负例（表型B）放在另一个。

可调超参数包括训练轮数、是否加正则化、学习率以及损失函数类型。

获得权重文件即可绘制热图。

命令行用法

```bash
python bi_class.py --wt_path WTPATH --pos POSPATH --neg NEGPATH --save_path SAVEPATH --length LENGTH
```

依次传递野生型序列路径、正例路径、负例路径、保存路径、序列长度。

### 回归任务

准备两份.fasta文件，分别为野生型序列和训练数据，训练数据须将所有输入数据保存为一个.fasta文件，同时在每个样本后补上表型标签，形如：

\> name1 label=1.00
ATCGATCGATCG......
\> name2 label=1.05
AGTCAGCTACGT......

注意label=1.00之间不要空格。其余同二分类任务。

```bash 
python regression.py --data DATAPATH --save_path SAVEPATH --length LENGTH
```
依次传递野生型序列路径、数据集路径、保存路径、序列长度。

### 预测功能

```bash
python predict.py --weight WEIGHTPATH --seq SEQUENCE --type DNA or RNA or PROTEIN
```

## 可视化方法

权重文件形状为 $c\times l$ ，主要可视化方式选择热图，横轴是位置，纵轴是20种氨基酸，每个格点的颜色深浅表征了该位置的该种氨基酸对表型的贡献或影响。但是这个方法存在问题，如果直接对权重的数值进行作图，将会各种颜色混杂、非常混乱，极为不直观，需要选择一个参考值进行去中心化。
参考值主要可以选择野生型序列的对应参数，或者额外引入“\*”。可以二者都试一下，按实际效果选择。

## 评价

目前已有的结果显示，该方法在几个数据集的关键位点定位的试验中（包括实验室的小数据集和诸如gb1数据集的大生物数据集）表现出近于MAVE-nn https://github.com/jbkinney/mavenn 的效果。同时，本项目环境要求较低，较为简洁干净，仅需Pytorch和一些基本库即可跑通。

但该项目仍存在一些问题，后续需要完善和解决：

- 全变分正则化的数学含义及其生物可解释性；
- 每个位点独立对表型贡献是否合理；
- 无法处理不等长的输入（对齐后，较短的氨基酸中间存在“-”），因此只能用野生型序列补齐，是否有更好的解决方法。

## 数据

项目中给出gb1蛋白数据集，约50,000个样本，可用于跑通模型。由于数据集较大，故压缩为.zip文件上传。
